## Prompt 8: The Accessibility Auditor

You are an Accessibility Specialist at Apple, ensuring designs work for everyone.

Perform a comprehensive accessibility audit of:
`[DESIGN DESCRIPTION OR UPLOADED DESIGN]`

Audit against WCAG 2.2 Level AA.

1. PERCEIVABLE
   - Text alternatives for images
   - Captions/transcripts for media
   - Avoid color-only messaging
   - Contrast targets:
     - Normal text 4.5:1
     - Large text 3:1
     - UI components 3:1
   - Resize text up to 200% without breakage
   - Avoid images of text

2. OPERABLE
   - Full keyboard operability
   - No keyboard traps
   - Skip links for repetitive content
   - Descriptive page titles
   - Logical focus order
   - Clear link purpose
   - Search/navigation/sitemap where applicable
   - Descriptive headings and labels
   - Visible focus indicators (minimum 2px, 3:1 contrast)
   - Pointer gestures alternatives
   - Reduced-motion option and no autoplay audio
   - Touch targets minimum 44×44 CSS px

3. UNDERSTANDABLE
   - Language identifiers
   - Consistent component behavior
   - Clear error identification and suggestions
   - Legal/financial/data error prevention
   - Contextual help

4. ROBUST
   - Valid HTML/markup
   - Name, role, value for components
   - Status messages with ARIA live regions

5. MOBILE-SPECIFIC
   - Rotation support
   - Input modality coverage
   - Reachability for thumb zones

6. COGNITIVE ACCESSIBILITY
   - Reading level target roughly Grade 8
   - Stable navigation
   - Plain-language errors
   - Extended/removed time limits
   - Flashing content below 3 flashes/sec

Deliverables:

- Pass/fail checklist by criterion
- Violations with location and severity
- Remediation recommendations
- Accessibility statement template
- QA testing checklist
- Screen-reader navigation flow
